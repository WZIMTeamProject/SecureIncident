# Project Vision

## Overview
SecureIncident is a web-based incident and problem reporting platform that enables organizations to manage, track, and resolve internal incidents in a structured, auditable way.

## Current State
- **Age**: ~3 weeks (started April 2026)
- **Status**: Early-stage MVP development
- **Users**: Internal use — team members and managers within organizations
- **Tech Stack**: React 19 + TypeScript (frontend), FastAPI + SQLAlchemy (backend), PostgreSQL/MySQL

## Purpose
Organizations face challenges tracking and resolving internal incidents in a consistent, auditable manner. SecureIncident provides a structured platform where teams can report incidents, assign responsibility, monitor resolution progress, and maintain a complete audit trail — replacing ad-hoc email chains or spreadsheets.

## Goals (Next 5-6 Weeks)
- Complete and integrate the frontend application (login, dashboard, incident management views)
- Merge backend API and database models into the main branch
- Deploy the full-stack application to AWS
- Implement an end-to-end CI/CD pipeline for automated testing and deployment
- Achieve a working MVP covering the core incident lifecycle: creation → assignment → resolution

## Evolution
The project started with domain specification and API design (schema-first approach), followed by parallel development of frontend scaffold, backend API, and database models across separate feature branches. The immediate objective is full integration and AWS deployment within the 5-6 week project timeframe.
